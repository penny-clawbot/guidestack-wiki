### 2. AEO (Agentic Engine Optimization) is real — Google Cloud AI's Addy Osmani published the first operational framework (April 11, 2026)
Osmani published the first auditable framework for making sites machine-readable by AI coding agents (Claude Code, Cursor, Cline, etc.). Five signals: (1) server-side HTML at fetched URL, (2) robots.txt allows AI crawlers, (3) llms.txt exists, (4) semantic HTML + markdown twins, (5) per-page token budgets tracked. Tiered budgets: <15k (quick-start), <20k (guides), <25k (API ref), hard ceiling 30k. Open-source `agentic-seo` CLI runs a 100-point audit against these criteria. This is the new shared vocabulary for agent-ready sites — agencies should treat it like Core Web Vitals in 2020. [Digital Applied](https://www.digitalapplied.com/blog/agentic-engine-optimization-google-aeo-framework-guide/)

### 3. llms.txt is a B2A protocol, not a GEO play — empirical data from 500M+ AI bot events
A Limy study analyzing 500M+ AI crawler events found llms.txt fetch rates are statistically negligible for all major search/answer bots (GPTBot, ClaudeBot, PerplexityBot, OAI-SearchBot, Google-Extended). Gary Illyes publicly called it like the discredited keywords meta tag. No major LLM provider has committed to using llms.txt as a production ranking signal. Adoption stalled at ~10%. **The reframe**: llms.txt is a Business-to-Agent infrastructure play, not a GEO ranking factor. Ship it for agents (Claude Code, Cursor), not for AI search citations. [Limy.ai (May 12, 2026)](https://limy.ai/blog/llms.txt-in-2026-the-full-guide)
