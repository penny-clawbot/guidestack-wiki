## 🔴 Breaking Changes

### 1. FAQ Rich Results RETIRED — May 7, 2026
Google officially retired FAQ rich results after 9 years. Any `FAQPage` JSON-LD schema currently deployed is **inert** — it produces zero SERP lift. It won't hurt your site, but it's dead weight consuming crawl budget.  
**Action:** Audit all deployed FAQPage schemas. Remove or repurpose them as `HowTo` or `QAPage` if the content warrants it. Do not build new FAQPage markup.

### 2. Universal Cart Feed-Layer Gate — Google I/O 2026
E-commerce sites targeting the Buy button in Google AI Mode need to understand: **schema.org Product markup is no longer the gate**. The actual eligibility path is:
- Merchant Center `native_commerce` feed attribute
- Capability server profile at `/.well-known/ucp`

This is a two-pipeline architecture. Product schema alone will not unlock AI Mode purchasing.
**Action:** For PSEO network e-commerce sites, document this as a Phase 2 requirement. Not urgent for informational sites.

### 3. Google Search Redesign — Biggest in 25 Years — May 19, 2026
Google I/O 2026 shipped the most significant Google Search UI change in its history:
- AI Mode is now default, with Gemini 3.5 Flash as the new model
- AI Mode surpassed **1B monthly users**; queries double every quarter
- The search box was completely redesigned with AI-powered suggestions and agent capabilities
- **Search agents** now launchable directly from the search box — AI does multi-step tasks on behalf of users

**Impact:** Zero-click searches are accelerating. Sources show 93% of AI Mode queries end without a click. This makes GEO citation the primary traffic path for informational content.

## 🟡 Key Developments

### 4. AI Crawler Behavior: 30-Day Site Log Study (Apr 26, 2026)
Real server log data across 12 sites (30 days) confirms:
- **GPTBot** = most aggressive at ~4,200 hits/site/day
- **ClaudeBot** = ~1,800 hits/site/day
- **PerplexityBot** = ~980 hits/site/day (only when users ask)
- **All major AI bots respect robots.txt 100%** — no violations observed

Bottom line: robots.txt directives for AI crawlers are being honored. No evidence of rogue crawling.

### 5. llms.txt — Not a Ranking Factor (May 12, 2026)
Analysis of 515M AI bot traffic events confirms: **llms.txt is not a GEO ranking factor**. It's a B2A (Business-to-Agent) distribution channel — useful for AI agent discovery, but it will not boost citations or rankings directly. The window to establish these files early is still open.

### 6. schema.org v30.0 + March 2026 Core Update
Google's March 2026 core update explicitly named **scaled content abuse** as a violation target. Structured data strategy has shifted — schema now primarily unlocks AI Mode citations and entity recognition rather than traditional rich results.

### 7. Google I/O 2026 Structured Data: Universal Cart + FAQ Retirement
Post-I/O schema landscape:
- `FAQPage` → **retired** (inert)
- `HowTo` → still eligible
- `Product` + `Offer` → still relevant for traditional e-commerce but **not sufficient** for Universal Cart
- `native_commerce` Merchant Center attribute → the new e-commerce AI gate

## 📋 Action Items for PSEO Network Template

1. **robots.txt** — Already has Agentics-Manifest directives (`Agentics-Manifest`, `Agentics-Version`, `Agentics-Capabilities`) which are **non-standard extensions**. These should be removed or replaced with the standard `/.well-known/agentsctl.json` approach if a real agentic crawl management standard emerges. Flag as experimental.

2. **llms.txt rel attribute** — BaseLayout.astro has `<link rel="llms-txt">` but the spec uses `llms.txt` as the URL path (e.g., `/llms.txt`). The `rel="llms-txt"` attribute is non-standard. The actual mechanism is just the URL path — no special `rel` value is defined in the spec. This should be corrected to just link to the file, not declare a new link relationship.

3. **FAQPage schema** — Any FAQ content components in the template should be audited. If FAQPage JSON-LD is in the template or components, remove it. Replace with `QAPage` or `HowTo` if Q&A content is genuinely useful.

4. **E-commerce schema stack** — For sites in the `/best/` or `/compare/` categories (which are effectively product comparison pages), the template should include a Product + Offer + AggregateRating schema stack. This is already mentioned in the agentic commerce guidance but not currently in the template.
